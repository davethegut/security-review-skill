## First-Pass Findings

### [HIGH] Acknowledge script has no max_docs cap

- **Checklist ref**: 2.3, 4.4
- **Location**: `alert-triage/scripts/acknowledge-alert.js:85-94`
- **Description**: `updateByQuery` uses `conflicts: 'proceed'` but no `max_docs`. The SKILL.md scopes by agent ID and time range, but the script has no upper bound.
- **Risk**: Wide time range or multi-host attack matches hundreds/thousands of alerts with no cap.
- **Remediation**: Add `--max-docs` parameter with default cap.

### [HIGH] KQL injection mitigation is documentation-only

- **Checklist ref**: 6.2
- **Location**: `SKILL.md:298-305` vs `alert-triage/scripts/run-query.js:49`
- **Description**: SKILL.md documents validation rules. The query script passes input directly to `query_string` with `analyze_wildcard: true` — no validation in code.
- **Risk**: Agent ignores validation instructions, attacker-controlled values inject arbitrary KQL.
- **Remediation**: Add server-side validation in the query script.

### [HIGH] Error messages may expose cluster URLs

- **Checklist ref**: 7.3
- **Location**: `acknowledge-discovery.js:91-94`, `shared/es-client.js`
- **Description**: Error output includes `result.details` which may contain internal URLs, index names, and cluster structure. Agent transcripts persist on disk.
- **Risk**: Internal cluster topology exposed in retained transcripts.
- **Remediation**: Redact host/URL from error output, preserving error type for debugging.

### [MEDIUM] Space ID not validated before URL construction

- **Checklist ref**: 6.5
- **Location**: `shared/kibana-client.js:49-55`
- **Description**: `getBasePath` inserts space ID into URL path with no validation. Malformed space ID could cause path traversal.
- **Remediation**: Validate with `/^[a-zA-Z0-9_-]+$/`.

### [MEDIUM] Batch operation does not propagate failure to exit code

- **Checklist ref**: 5.5
- **Location**: `case-management/scripts/case-manager.js:107-138`
- **Description**: Exits with code 0 even if some attachments failed. Verification gate mitigates if followed.
- **Remediation**: Exit with code 1 if `failed > 0`.

### [MEDIUM] CLI accepts shell-expanded input

- **Checklist ref**: 6.1
- **Location**: `case-management/scripts/case-manager.js:11-33`
- **Description**: `parseArgs` reads `process.argv` after shell expansion. Single-quote guidance mitigates if followed.
- **Remediation**: Add `--json-stdin` mode for structured input.

### [MEDIUM] Node.js prerequisite says v18+ but code requires v21.7+

- **Checklist ref**: 7.6, 9 (drift)
- **Location**: `SKILL.md:54,56` vs `shared/kibana-client.js:6`
- **Description**: `process.loadEnvFile()` was added in Node 21.7.0. On Node 18-20, the `try/catch` swallows the error and `.env` is never loaded. Scripts fail with misleading error.
- **Remediation**: Either raise the prerequisite to Node 21.7+ or document the workaround.
